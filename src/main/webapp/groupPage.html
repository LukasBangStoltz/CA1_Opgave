Gruppekontrakt

Man pusher aldrig til master, men altid til sin egen branch, hvorefter man kan lave en pull request, og derefter merge det med master. ALTID!!! mind resten af gruppen om at teste i små steps af gangen. Lav små push hele tiden, så tingene sikres at det virker.

Fremgang →

  • Lav en ny branch git checkout -b ‘bran chname’
  • mvn clean test på branch inden add og commit
  • Add til din nye branch git acm
  • Push til din branch git push origin ‘branchName’
  • Lav pull request på github
